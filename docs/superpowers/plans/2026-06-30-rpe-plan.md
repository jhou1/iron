# RPE Session Tracking Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow users to optionally record a Rate of Perceived Exertion (RPE) from 1-10 for their training sessions.

**Architecture:** We are adding a nullable `rpe` integer column to the `training_sessions` table, propagating it through the models, export schemas, and displaying it via a new input on the Log Entry TUI screen and a badge on the History TUI screen.

**Tech Stack:** Rust, SQLite (rusqlite), Ratatui.
